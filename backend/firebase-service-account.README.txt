Firebase Admin (recommended for Google sign-in on the server)
================================================================

If you see "Invalid Google token" after fixing the 502 error, the server must verify
Firebase ID tokens correctly.

EASIEST FIX (recommended)
-------------------------
1. Firebase Console → Project settings → Service accounts
2. Click "Generate new private key" → download the JSON file
3. Save it as:  backend/firebase-service-account.json
4. Restart the backend (npm run dev)

Never commit this file to git.

ALTERNATIVE
-----------
Set env FIREBASE_SERVICE_ACCOUNT_PATH to the full path of that JSON file,
or FIREBASE_SERVICE_ACCOUNT_JSON to the raw JSON string (not ideal for .env).

WITHOUT SERVICE ACCOUNT
-----------------------
The backend falls back to Google's tokeninfo API using POST (not GET).
If it still fails, set in backend/.env:

  FIREBASE_PROJECT_ID=gloginbookstore

If tokeninfo reports a different "aud" than your project id, add:

  FIREBASE_ACCEPTED_AUDIENCES=the-value-tokeninfo-shows
